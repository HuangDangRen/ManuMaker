§<html dir="ltr"><head></head><body contenteditable="true"><h3><font face="Segoe UI" size="4">Transferring the Gallery's information to HTML:</font></h3><p><font face="Segoe UI">To understand the reasons behind the following instructions, the underlying structure of the exported slides in HTML form must be understood.</font></p><p></p><ul><li><font face="Segoe UI">During the export process, all the data of a slide is transferred into a HTML template.&nbsp;<br><br></font></li><li><font face="Segoe UI">Every slide type (Title, Chapter, Instructional with picture(s), Text only) has it's own template file, which is located in the installation directory under <i>app/resources/printable_html/...<br><br></i></font></li><li><font face="Segoe UI">The templates (all HTML templates ManuMaker uses for display, print and warnings) are <b>intended </b>to be replaced, if a different design is needed to e.g. conform to a company's style guide. <br>To replace them, please refer to the source code of the respective base files in the directory named above and make sure any replacements contain the same root elements<br><br></font></li><li><font face="Segoe UI">Templates can easily be created by using table calculation software for defining the colour, position and size of background and text, use the export / save function which generates the most plain html&nbsp;<br><br></font></li><li><font face="Segoe UI">Before you alter any of the html template files, make sure you create a backup first.&nbsp;<br><br></font></li><li><font face="Segoe UI">Each template defines the visual appearance:</font></li><ul><li><font face="Segoe UI">Minimum width of the page (defined by giving the template a white-coloured frame which the viewer doesn't see. otherwise the empty spaces would be shrunk together as HTML does not preserve whitespaces and conventional newlines)</font></li><li><font face="Segoe UI">fontsize and location of different text elements:</font></li><ul><li><font face="Segoe UI">This is achieved through the use of placeholder words</font></li><li><font face="Segoe UI">When the export-version of a slide is generated, the html template is loaded (as a String, using UTF-8 encoding) and all the placeholders are replaced by the relevant slide data via String.replace operations.&nbsp;</font></li><li><font face="Segoe UI">If a placeholder would be to appear twice, both instances would be replaced with the same data</font></li><li><font face="Segoe UI">If a placeholder could not be found, no exception will be thrown and the corresponding data will simply not appear in the exported manual</font></li></ul><li><font face="Segoe UI">Frames around elements and colour-scheme for the page&nbsp;<br><br></font></li></ul><li><font face="Segoe UI">The instructional page-templates all consist of the following elements:</font></li><ul><li><font face="Segoe UI">Header, which contains the slide number (All slides including chapter and title slide are counted, counting starts at <u><b>1</b></u>)&nbsp;</font></li><li><font face="Segoe UI">Title of the slide printed in large, bold letters</font></li><li><font face="Segoe UI">Information data (Image, Text, or both), which usually makes up the largest portion of the slide</font></li><li><font face="Segoe UI">the placeholder for the information data is followed by a pagebreak (Not part of the slide-specific template, every slide is loaded into the page-template <i>TMP_Page.html</i>)<br><br></font></li></ul><li><font face="Segoe UI">There are two key aspects to be considered when making a slide printable:&nbsp;</font></li><ul><li><font face="Segoe UI">Keep contents to a minimum width (which is not a concern unless the template-files are modified)</font></li><li><font face="Segoe UI">Keep the amount of information small enough to fit on its page, without pushing the page-break over the edge to the next page.</font></li></ul></ul><p><br></p>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;</body></html>&nbsp;&nbsp;
